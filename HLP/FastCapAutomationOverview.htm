FastCap2 Automation Overview

FastCap2 provides a Windows Automation interface which allows a remote application to fully control FastCap2.

For example, a Visual Basic script inside an MS Windows Excel sheet can launch FastCap2 in background, make it perform some simulation, then retrieve the results and show them in some cells - and all without even showing up the FastCap2 window!

The Automation Object

FastCap2 can be controlled by any Automation-capable application. The automation object which must be created to control FastCap2 is named FastCap2.Document. This name is defined in the Windows Registry at installation time or, if removed from the Registry, any time FastCap2 is run.
