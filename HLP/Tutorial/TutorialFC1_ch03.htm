Understand basic FastCap2 input file syntax

Browse the input file

  1. If the FastCap2 View is not visible, bring it in foreground
  2. Examine the first line of the view. The line begins with a '0', then contains a description of the content of the file. Every FastCap2 file must begin with such a line.
  3. Examine the following lines. All lines begin with the letter 'T'. This means that the line describes a triangle. Then follows a conductor name, 'sphere', and a triplet of 3D coordinates, specifying the triangle corners, as highlighted in the following figure.

  4. Go to the end of file. Notice that there is no need of any end of file specifier.

Add comments

To add a comment, simply begin a line with a '*'. The line will be recognized as comment and highlighted in color.
  1. Add an asterisk '*' at the very beginning of lines 5, 6, 7, 8, as shown in the following figure (you can easily find the lines looking at the row counter on the status bar, which shows the current cursor position).

  2. Right click with the mouse on the FastCap2 view and from the pop-up menu choose Update Model
  3. Examine the FastModel View. You should notice that four triangles disappeared from the sphere surface. If you cannot see it, try rotating the sphere.
  4. Remove the asterisks and update the model. The sphere shoud appear complete again.

Add patches

As 'T' keywords are used to specify triangular patches, the 'Q' keyword is used to describe a quadrilateral panel.
  1. Go to the end of the FastCap View
  2. Add the following line after the last 'T' statement

    Q plane -1 -1 0 1 -1 0 1 1 0 -1 1 0

  3. Right click with the mouse on the FastCap2 view and from the pop-up menu choose Update Model
  4. Examine the FastModel View. You shoud see a quadrilateral plane cutting the sphere in two halves.
  5. Delete or comment the last line you added and update the model. The plane should disappear.
