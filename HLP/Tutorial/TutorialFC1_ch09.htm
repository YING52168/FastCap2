Define dielectric interfaces

Dielectric interface panels are specified using the same syntax used for conductor panels. It is the list file that specifies if the panels must be treated as conductors or dielectric interfaces.

  1. Load the sample file coated_sph.lst into FastModel. You should see two concentric spheres in the FastModel view.

  2. Examine the list file in the FastCap2 View.

    The first non-comment line begins with a 'C' character. This line specifies that the file sphere2.qui is used to define a conductor. The conductor is surrounded by a dielectric medium whose dielectric constant is 2.0. Moreover, the coordinates of the panels described in the sphere2.qui file have a 3D offset of (0.0, 0.0, 0.0), that is, no offset.

    The second non-comment line begins with a 'D' character. This line specifies that the file sphere_big2.qui is used to define a dielectric interface. This is the outer shell containing the conductive sphere. The dielectric medium on one side of the interface has dielectric constant 1.0, while the medium on the other side has dielectric constant 2.0. The coordinates of the panels described in the sphere_big2.qui file have a 3D offset of (0.0, 0.0, 0.0), that is, no offset. In the end, the last triplet of numbers, together with the '-' sign, defines the reference point for the 'in' side of the interface. In this case, the command means that the point (0.0, 0.0, 0.0) lies in the region of space filled with dielectric medium with constant 2.0.

  3. Launch a FastCap simulation.
  4. Examine the result. The value for the capacitance should be 0.1478 nF, well within 1% of the 0.14835 nF value expected by Gauss's Law.
