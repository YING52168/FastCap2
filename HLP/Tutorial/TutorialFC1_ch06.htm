Refine the geometry to achieve greater accuracy

The capacitance of a sphere

The sphere is a geometrical object for which a closed-form formula for the calculation of capacitance exists. By Gauss's law, a conducting sphere with one meter radius in free space has a capacitance of 4πε0 Farad, that is, about 0.111265 nF.

By comparing this result with FastCap2's, which is 0.1058 nF, we have about 5% error in the capacitance value. FastCap2's declared accuracy is 1%, but only provided with a suitable input discretization.

A refined sphere

  1. Load in FastModel the input file named sphere2.qui, which is located in the same directory of sphere1.qui.
  2. Examine the sphere in the FastModel panel. You'll see that this is a more refined model of a sphere.
  3. Launch FastCap2 on the new input file, just like you did with the previous file.

Checking the result

  1. Look at the result at the end of FastCap2's output window. The calculated capacitance is 0.1098 nF, which is within 1% of the correct value of 0.111265 nF.
  2. In this particular case, the correct value of the capacitance was known beforehand trough an analytical formula. But how, in general, the user can be sure to have reached 1% accuracy in the results? To answer this question, load the file sphere3.qui, which is located in the same directory of sphere1.qui and sphere2.qui, into FastModel.
  3. Examine the sphere in the FastModel panel. You'll see that this is an even more refined model of a sphere.
  4. Launch FastCap2 on the new input file, just like you did with the previous files.
  5. Check the new result, which is 0.1109 nF, even closer to 0.111265 nF. However, what is important here is that the capacitance calculated with the file sphere3.qui is within 1% of the capacitance calculated with the file sphere2.qui. This means that we have reached convergence, even if we had not known that the correct result was 0.111265 nF.
